# About this guide

**Audience:** Maintainers, contributors, and agents implementing or reviewing engine behavior.

**Scope:** Capabilities, design constraints, and architecture pointers — not step-by-step consumer tutorials. End-user usage lives in [client guide](../client/index.md); repository workflow lives in [developer guide](../developer/index.md).

**As-built vs direction:** Shards at this level describe **what ships today** (packages, parsers, syntax surfaces). Active design, ADRs, and V2 goals live under [Architecture](./architecture/index.md). When behavior and direction diverge, trust the codebase for implementation and the architecture shards (especially [V2 design goals](./architecture/v2_design_goals.md)) for intent.
